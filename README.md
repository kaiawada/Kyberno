# Kyberno
FastAPIとMQTTを用いた、拡張性の高い水耕栽培・環境制御システム。

## プロジェクト背景,経緯
2023年にハウス農家で実務に従事した際、「現場の環境制御をソフトウェアやIoTなど活用できることがあるんじゃないか」と感じ、またエンジニアとして興味の引っ掛かかりどころなのではと感じたのが原点です。

### 1.現場の課題と最初の挫折
「世界中のどこからでも遠隔制御できるシステム」という目標をぼんやりと決め、調べながら進めようとスタートしました。
当初、以前本を見ながら作成したWebアプリがあるという理由だけでですが Django を用いました。 Web アプリと給水ポンプの自作に挑戦しました。しかし、当時はインフラ知識や通信プロトコルはもちろんデバイスの知識や理解は雰囲気程度の理解しかなく、かろうじて同ネットワーク内での給水ポンプの遠隔操作はできましたが様々な問題がありました。

###　2. 失敗分析と再構築
それまでの試行をもとに「なぜ動かないのか」を調べ、その中で少しずつ何がわからないかの理解しようと進めていきました。サーバー制約やプロトコル選択の誤りなど多くの課題が明確になりました。この経験を糧に、現在のそれなりに最適なのではというシステム構成のアイデアへと辿り着きました。

### 3. Project "Kyberno"
遠隔地からの確実な環境制御と、柔軟な機能拡張に対応できるシステムの構築を目指します。悲しいかなやりたいことは想像程度にあるけど、世に出ているデバイスをいきなり使う知識も、そもそも資金もないので「ないものは自作」という方針で試行錯誤しながら最適な形を模索しています。自作デバイス（ESP8266等）からスタートし、最終的には世の中にある様々な汎用センサーや機器をプラグアンドプレイで組み込めるような、自由度の高いプラットフォームを目指しています。

## 構成
- **Backend**: FastAPI
- **Messaging**: MQTT (Mosquitto)
- **Edge Devices**: ESP8266 / NodeMCU
- **Infrastructure**: Raspberry Pi 4B
